# Product Context

## Why this project exists

To provide a controlled environment for testing, debugging, and learning complex Java backend concepts in a SmartOSC training context.

## Problems it solves

- Reproducing elusive concurrency bugs.
- Demonstrating the impact of poor transaction management.
- Providing a standard blueprint for Spring Security and Architecture.

## User Experience Goals

- Developers should be able to run and observe bug scenarios easily.
- Clear logs and documentation for each technical decision.
- Production-grade code structure as an example.
