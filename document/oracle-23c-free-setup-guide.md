## 1. Kiến trúc Oracle 23c Free (Architecture Context)

Oracle 23c (và các bản từ 12c trở đi) sử dụng kiến trúc **Multitenant**:
Oracle 23c (and 12c+ versions) uses **Multitenant** architecture:

- **CDB (Container Database)**: Cơ sở dữ liệu gốc (thường tên là `FREE`).
- **PDB (Pluggable Database)**: Cơ sở dữ liệu con gắn vào CDB (thường tên là `FREEPDB1`).
- **Lưu ý**: Mọi dữ liệu ứng dụng phải nằm trong **PDB**.
- **Note**: All application data must reside within the **PDB**.

## 2. Thiết lập User hệ thống (System User Setup)

Đăng nhập bằng account **SYS** với quyền `SYSDBA` để tạo môi trường.
Log in as **SYS** with `SYSDBA` privileges to set up the environment.

### SQL Queries (Run as SYS):

```sql
-- 1. Chuyển sang context của PDB
-- 1. Switch to the PDB context
ALTER SESSION SET CONTAINER = FREEPDB1;

-- 2. Tạo User cho ứng dụng
-- 2. Create the Application User
CREATE USER backend_debug IDENTIFIED BY root123;

-- 3. Cấp quyền DBA cho buổi demo (Đảm bảo không lỗi Permission)
-- 3. Grant DBA permissions for the demo (Ensure no Permission errors)
GRANT CONNECT, RESOURCE, DBA TO backend_debug;

-- 4. Cấp hạn mức lưu trữ (BẮT BUỘC để Create Table/Insert)
-- 4. Grant storage quota (MANDATORY for Create Table/Insert)
ALTER USER backend_debug QUOTA UNLIMITED ON USERS;

-- 5. Mở khóa account
-- 5. Unlock the account
ALTER USER backend_debug ACCOUNT UNLOCK;
```

## 3. Cấu hình Spring Boot (application-dev.yaml)

Cập nhật file cấu hình để kết nối tới user vừa tạo.
Update the configuration file to connect to the newly created user.

```yaml
spring:
  datasource:
    url: jdbc:oracle:thin:@//localhost:1521/FREEPDB1
    username: backend_debug
    password: root123
    driver-class-name: oracle.jdbc.OracleDriver
  jpa:
    hibernate:
      ddl-auto: none # Sử dụng script Flyway/SQL thủ công
    properties:
      hibernate:
        dialect: org.hibernate.dialect.OracleDialect
```

## 4. Chuyển đổi Schema PostgreSQL sang Oracle (Schema Conversion)

### Những lưu ý khi chuyển đổi (Conversion Highlights):

- **`BIGSERIAL`** (Postgres) ➔ **`NUMBER(19) GENERATED BY DEFAULT AS IDENTITY`** (Oracle).
- **`VARCHAR`** ➔ **`VARCHAR2`**.
- **`TEXT`** ➔ **`CLOB`**.
- **`updated_at` Trigger**: Oracle yêu cầu Trigger riêng để tự động cập nhật timestamp (không dùng `ON UPDATE CURRENT_TIMESTAMP` trực tiếp như MySQL).
- **`updated_at` Trigger**: Oracle requires a dedicated Trigger to auto-update timestamps (no direct `ON UPDATE CURRENT_TIMESTAMP` like MySQL).

### Mã SQL mẫu (Sample SQL):

```sql
CREATE TABLE tbl_user (
    id         NUMBER(19) GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username   VARCHAR2(255) NOT NULL UNIQUE,
    email      VARCHAR2(255) NOT NULL UNIQUE,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP NOT NULL
);

-- Trigger cập nhật timestamp
CREATE OR REPLACE TRIGGER trg_user_updated
    BEFORE UPDATE ON tbl_user
    FOR EACH ROW
BEGIN
    :NEW.updated_at := CURRENT_TIMESTAMP;
END;
/
```

## 5. Mẹo cho buổi Presentation (Presentation Tips)

Khi trình bày với Leader, hãy nhấn mạnh các điểm sau:
When presenting to the Leader, emphasize the following points:

1. **Security**: Tại sao không dùng user `SYS` hay `oracle`? Vì chuẩn ngành (Banking/Fintech) yêu cầu **Principle of Least Privilege** (Quyền tối thiểu).
2. **Security**: Why not use `SYS` or `oracle`? Because industry standards (Banking/Fintech) require the **Principle of Least Privilege**.
3. **Storage**: Tại sao phải cấp `QUOTA`? Oracle quản lý tài nguyên rất chặt chẽ, việc cấp quyền tạo bảng không đồng nghĩa với việc cho phép dùng dung lượng ổ cứng.
4. **Storage**: Why grant `QUOTA`? Oracle manages resources strictly; granting table creation rights doesn't automatically mean allowing disk space usage.
5. **Consistency**: Việc cấu hình `ddl-auto: none` và dùng SQL script giúp đảm bảo tính nhất quán giữa môi trường Dev và Production.
6. **Consistency**: Configuring `ddl-auto: none` and using SQL scripts ensures consistency between Dev and Production environments.
